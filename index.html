ℹ️ Comment lire les signaux (fiabilité, IC, Neff)
Signal basé sur l'intervalle de confiance à 95 % (IC95) :
  • ACHAT — borne basse de l'IC > 55 % (haussier même dans le pire cas)
  • VENTE — borne haute de l'IC < 45 %
  • NEUTRE — l'IC traverse 50 % (incertitude statistique)
  • INSUFFISANT — liquidité totale < 20 k$ (pas assez d'argent en jeu pour faire confiance au prix)
Fiabilité (basée sur l'erreur-type SE du score haussier) :
  • Élevée — SE < 4 pts → IC resserré, marchés d'accord, signal robuste
  • Moyenne — SE entre 4 et 8 pts → à prendre avec précaution
  • Faible — SE ≥ 8 pts ou liquidité trop basse → ne pas utiliser pour décider
Neff (effective sample size de Kish) : nombre de « marchés indépendants équivalents » qui pèsent sur le score. Si 1 gros marché domine, Neff ≈ 1 même avec 5 marchés au total.

Horizon prédictif filtre les marchés par endDate : « 1 mois » garde les marchés qui expirent dans 7 à 45 jours — c'est la fenêtre où Polymarket reflète la tendance attendue à moyen terme.

Indicateurs techniques complémentaires (cryptos uniquement) :
  • 💰 Funding rate (Binance Futures) : coût toutes les 8 h payé entre longs et shorts. Indicateur contrarian : > 0.05 % = trop de longs → risque de correction baissière ; < −0.03 % = trop de shorts → risque de squeeze haussier.
  • 📈 RSI 14 jours calculé sur les bougies daily Binance. < 30 = survente (bullish contrarian), > 70 = surachat (bearish contrarian).
  • 🟢/🔴 Signal composite : moyenne pondérée Polymarket (poids 2) + Funding (1) + RSI (1). Déclenché dès que 3 sources sur 4 s'alignent. Le nombre « X/Y sources d'accord » est votre indice de fiabilité prédictive composite.
  • 😨 Fear & Greed Index (bandeau en haut) : contexte global crypto. Un achat en « peur extrême » est historiquement plus rentable qu'en « cupidité extrême ».

Méthodologie

3 API officielles Polymarket :
• 🟢 Gamma API — gamma-api.polymarket.com/markets et /events — liste des marchés actifs + probabilités. Public, pas d'auth.
• 🔵 CLOB API — clob.polymarket.com/prices-history — historique de prix réel de chaque token YES/NO (intervalles 1h/1d/1w/max). Reconstitue l'évolution multi-jours.
• 🟣 Data API — data-api.polymarket.com/trades — statistiques agrégées en appoint.

Classification : chaque marché est classé haussier (« reach », « above », « exceed »…) ou baissier (« below », « crash », « fall »…) par analyse du titre. La probabilité du résultat "Yes" donne la conviction du marché.
Score : moyenne pondérée par le log du volume (bullish = p, bearish = 1-p). Score final ∈ [0 ; 1]. Signal : > 0.58 = ACHAT, < 0.42 = VENTE, sinon NEUTRE.
Historique : CLOB pour la vérité des prix Polymarket + snapshots locaux pour votre vue agrégée jour après jour. Export/import JSON pour sauvegarde.
Limites : cet outil reflète l'opinion des parieurs Polymarket — ce n'est pas un conseil financier.
